# 0002. Правила Шарової Архітектури

| Атрибут | Значення |
| :--- | :--- |
| **Статус** | Прийнято |
| **Дата** | 05.12.202णीय |
| **Рішення** | Використовувати класичну тришарову архітектуру: **API → Service (Use Cases) → Domain**. |
| **Структура** | api/ — контролери/роути (Express), service/ — сервіси/UseCase, domain/ — доменні моделі (Item, Order). |
| **Залежності** | Дозволено: `API` ➡️ `Service` ➡️ `Domain`. Заборонено: Залежності знизу вгору (наприклад, `Domain` не залежить від `Service`). |
| **Заборонені зв'язки** | **API не повинен звертатися напряму до бази даних** (наприклад, db-connector). **Вся бізнес-логіка** (валідація, інваріанти) повинна бути реалізована у шарах `Service` або `Domain`. |
