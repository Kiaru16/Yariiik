# 0003. Стиль API та Формат Помилок

| Атрибут | Значення |
| :--- | :--- |
| **Статус** | Прийнято |
| **Дата** | 05.12.2025 |
| **Рішення** | Встановлюється стиль **RESTful API** та вимога повної відповідності специфікації **OpenAPI (docs/api/openapi.yaml)**. |
| **Формат Помилок** | Усі помилки повинні повертатися у єдиному форматі `ErrorResponse` (як описано в OpenAPI). |
| **Політика Кодів** | HTTP-коди мають використовуватися для комунікації статусу (200 OK, 201 Created, 400 Bad Request, 404 Not Found, 500 Internal Server Error). Коди 4xx повинні включати внутрішній `code` (наприклад, `ITEM_NOT_FOUND`). |
