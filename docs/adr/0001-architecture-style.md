# 0001. Вибір Архітектурного Стилю: Модульний Моноліт

| Атрибут | Значення |
| :--- | :--- |
| **Статус** | Прийнято |
| **Дата** | 05.12.2025 |
| **Контекст** | Проєкт "Mini-Shop" знаходиться на ранній стадії (MVP), команда складається з однієї особи. Головна мета — швидка імплементація бізнес-логіки та паттернів DDD. |
| **Рішення** | Використовувати **Модульний Моноліт** з чітким внутрішнім розділенням на домени (Catalog, Order). |
| **Альтернативи** | Пілот мікросервісів (відхилено). |
| **Наслідки** | **Позитивні:** Спрощене розгортання та тестування, швидкий старт, відсутність overhead'у на мережеву комунікацію. **Негативні:** Згодом може ускладнитися горизонтальне масштабування окремих модулів. |
| **Умови перегляду** | Перехід до мікросервісів буде розглянутий, коли: 1) Команда розробки зросте до 3+ осіб. 2) Виникнуть критичні вимоги до масштабування (наприклад, контекст "Обробка Замовлень" вимагатиме значно більше ресурсів, ніж "Управління Каталогом"). |
